
# Test Cases

## What is a Test Case?
A test case is a set of conditions or steps used to verify a specific functionality of an application.

## Components of Test Case
- Test Case ID
- Description
- Steps
- Expected Result
- Actual Result

## Types of Test Cases
- Functional  --- 👉 Focus: “Does it do what it should?”  (Verify that the system works according to requirements (business logic is correct).
- Negative --- 👉 Focus: “Does it fail correctly?”   (Test with invalid, incorrect, or unexpected inputs to check how system behaves.)
- Boundary --- 👉 Focus: “What happens at limits?”   (Test values at the edge limits of input fields.)
- UI Test Cases --- 👉 Focus: “Does it look and behave correctly?”  (Verify frontend appearance and behavior.)
